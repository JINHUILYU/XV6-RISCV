# xv6-riscv 使用教程（学习版）

## 1. 环境准备

1. 获取源码：
	- 上游仓库：https://github.com/mit-pdos/xv6-riscv
	- 或直接使用当前的仓库代码。
2. 安装工具链：
	- RISC-V 交叉编译工具链（如 `riscv64-unknown-elf-gcc`）
	- `qemu-system-riscv64`
	- 可选：`gdb`（调试用）

## 2. 基本运行

在仓库根目录执行：

```bash
make qemu
```

启动后会进入 xv6 的 shell（`sh`）。

退出 xv6：

1. 先按 `Ctrl+a`
2. 松开后再按 `x`

输入 EOF：（`Ctrl+d`）。

## 3. 在哪里设置“要运行的程序”

这部分是最关键的。

### 3.1 把程序打包进 xv6 镜像

用户态程序是否能在 xv6 里执行，取决于 `Makefile` 里的 `UPROGS`。

- 位置： [Makefile](Makefile#L125)
- 规则：每个用户程序都要以 `$U/_程序名` 的形式加入 `UPROGS`

例如你有 `user/hello.c`，就需要在 `UPROGS` 中包含：

```makefile
$U/_hello\
```

说明：

- `user/hello.c` 会被构建为 `user/_hello`
- 进入 xv6 后执行命令时写 `hello`（不是 `_hello`）

### 3.2 设置开机默认启动哪个用户程序

xv6 启动第一个用户态进程是 `init`，它默认拉起 `sh`。

- 位置： [user/init.c](user/init.c#L11)
- 当前默认：`char *argv[] = { "sh", 0 };`
- 真正执行点： [user/init.c](user/init.c#L30) 的 `exec("sh", argv);`

如果你希望开机直接运行自己的程序（例如 `mycat`），可把 `sh` 改成 `mycat`（并确保它已经在 `UPROGS` 里）。

## 4. 新增一个用户程序的完整流程

以新增 `user/hello.c` 为例：

1. 创建 `user/hello.c`，实现 `main`。
2. 在 [Makefile](Makefile#L125) 的 `UPROGS` 里加入 `$U/_hello\`。
3. 重新构建并运行：

```bash
make clean
make qemu
```

4. 在 xv6 shell 中执行：

```sh
hello
```

## 5. 常用命令

1. 普通运行：

```bash
make qemu
```

2. 启动并等待 GDB 连接：

```bash
make qemu-gdb
```

3. 清理构建产物：

```bash
make clean
```

## 6. 常见问题

1. 命令找不到（`exec xxx failed`）
	- 大概率是没有把程序加入 `UPROGS`。

2. 修改代码后运行结果没变化
	- 先 `make clean` 再 `make qemu`，确保镜像和程序都被重新生成。

3. 启动时报找不到 RISC-V 工具链
	- 检查交叉编译器是否在 `PATH` 中，或按 [Makefile](Makefile#L38) 的 `TOOLPREFIX` 规则配置。
